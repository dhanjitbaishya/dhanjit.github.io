Contact Management System

This Python-based project provides a command-line interface for managing contact information, including creating, updating, deleting, and listing contacts. Designed for ease of use, this application helps users efficiently manage contact details stored in a MongoDB database, making it a practical solution for contact data handling.

Key Features
  • Add New Contacts: Easily create and save new contacts with relevant details.
  • Update Contact Information: Modify existing contact information in the database.
  • Delete Contacts: Remove outdated or incorrect contacts.
  • List All Contacts: Display a list of saved contacts with their details, providing a simple way to view and retrieve information.
Technologies Used
  • Python: The core language for implementing functionalities and managing database interactions.
  • MongoDB: NoSQL database used for storing contact information, allowing flexible data storage.
  • PyMongo: Python library for MongoDB integration, enabling direct interaction with MongoDB from the Python environment.
Setup and Requirements

To run this project, you need to install the pymongo library. Use the following command on Windows: 

pip install pymongo
Sample Usage

Screenshots labeled CONM1 and CONM2 demonstrate examples of adding, updating, deleting, and listing contacts using this project.

Learning Outcomes

This project helped deepen my understanding of database management with MongoDB and Python's data-handling capabilities. It also enhanced my skills in CRUD (Create, Read, Update, Delete) operations and basic CLI application design.

Future Enhancements

For improved functionality, a GUI can be integrated using Python frameworks like Tkinter or PyQt. Additionally, adding search functionality or implementing user authentication would make the project more robust.

Express REST API

This project is a RESTful API built using Express.js to provide basic CRUD functionality for a hypothetical T-shirt service. It includes endpoints for retrieving T-shirt details and submitting new T-shirt orders, using Express.js to handle routing and JSON data parsing. The project demonstrates proficiency in back-end development and REST API creation.

Key Features
  • GET /tshirt: Returns a JSON object with a sample T-shirt item and its size.
  • POST /tshirt/:id: Accepts a T-shirt ID and a logo to customize the T-shirt, returning a customized message if the logo is provided.
  • JSON Parsing: Uses Express's built-in JSON parser to handle incoming JSON data in requests.
  • Error Handling: Returns an HTTP 418 error if the required "logo" field is missing in POST requests.
Technologies Used
  • Node.js: JavaScript runtime environment used for building the back-end server.
  • Express.js: Web framework for setting up routes and handling HTTP requests.
Setup and Requirements

To run this project, you need to install the necessary packages. Use the following command: 

npm install express
Usage

Start the server with node index.js and test the endpoints by visiting http://localhost:8080/tshirt or using a tool like Postman for POST requests.

Learning Outcomes

This project helped reinforce knowledge of Express.js, API design, and HTTP methods. It also provided insight into building RESTful services and handling request data efficiently.

Future Enhancements

Additional endpoints and database integration (e.g., MongoDB) could be added to expand functionality, enabling more detailed T-shirt customization and order management.
